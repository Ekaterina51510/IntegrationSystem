# IntegrationSystem
тестовое задание клиент-серверная программа
Распределенная система численного интегрирования

Описание проекта
Клиент-серверная система для вычисления интеграла функции f(x) = 1/ln(x) с использованием распределенных вычислений. Система поддерживает два различных метода численного интегрирования, как того требует задание.

Архитектура системы
- Сервер (IntegrationServer) - принимает параметры интегрирования от пользователя, распределяет задачи между клиентами пропорционально их вычислительным мощностям и суммирует результаты.
- Клиент 1 (IntegrationClient) - вычисляет интеграл методом прямоугольников.
- Клиент 2 (IntegrationClient2) - вычисляет интеграл методом трапеций.

Реализованные методы интегрирования:
- Метод прямоугольников (IntegrationClient)
- Метод трапеций (IntegrationClient2)

Сборка проекта
1.Откройте папку проекта в Visual Studio (2017 или новее)
2.Нажмите F5 для сборки и запуска
Сервер запросит:
Начало (a): нижний предел интегрирования
Конец (b): верхний предел интегрирования
Шаг (h): точность вычислений (например: 0.001)
Количество клиентов: сколько клиентов будет вычислять

Запустить:
1. Сначала запусти сервер (IntegrationServer)
2. Введи параметры: начало, конец, шаг, сколько клиентов
3. Потом запусти клиент (IntegrationClient или IntegrationClient2)
4. Клиент посчитает и вернет результат
Важно: Запускайте клиентов после того, как сервер начнет их ожидать!

Файлы проекта
Основные файлы:
IntegrationServer.cpp - главный сервер, распределяет задачи
IntegrationClient.cpp - клиент с методом прямоугольников
IntegrationClient2.cpp - клиент с методом трапеций
Вычислительные модули:
calculator.h, calculator.cpp - метод прямоугольников
calculator2.h, calculator2.cpp - метод трапеций
CMakeLists.txt - файл сборки CMake

Особенности
Методы интегрирования:
Прямоугольники - быстрый, простой метод
Трапеции - более точный, но требует больше вычислений
Распределение нагрузки:
Сервер делит интервал [a, b] между клиентами пропорционально их ядрам CPU.
Логирование:
После запуска создаются файлы:
server.log - действия сервера
client.log - клиент с прямоугольниками
client2.log - клиент с трапециями
